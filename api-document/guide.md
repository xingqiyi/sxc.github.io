# 接口文档使用说明

```
所有登录请求 采用post提交
测试地址 http://192.168.0.22:10011/capi.php
正式地址
SIGN_KEY : 20170215*&^%^$#@

请求参数 data = ??? (json 字符串)
data 为一个 json数据 其参数为
	c  // 请求操作code
	t  // 请求时间, 必须校准服务器时间
	d  //(json 序列化字符串)  需要请求的数据
	s  // 请求签名 $s = md5($c.$t.$d. SIGN_KEY);

如  data = {"c":1,"t":1486364446,"d":"{\"username\":\"15171505291\",\"key1\":\"65ad6b4ed916027dca32b318c4ce127b\"}","s":"367d15daf452ab55db18cfea4d428668"}
返回格式 (json 字符串)
	status 状态
	data 返回的数据
	错误:
	{"status":0,"info":"您没有登录,请先登录"}
	正确
	{"status":1,"info":"","data":{"rolename":"网吧管理员","sid":"123123","id":"8","nickname":"3","name":"15171505291"}}

所有请求(不包括登录)需要带默认参数
	sid:  //sessionid  //登录成功后返回
	id:  // 用户id  登录成功后返回
如
        {"c":2,"t":1486364605,"d":"{\"id\":8,\"sid\":\"123123\"}","s":"1e93c8a6b2e6fddd22e6c96e44c57261"}

以下api 返回数据 只描述 data里面的参数

所有 id 都是整型，只是在传递的时候都是以字符串传递
```
